# Presenter notes - Session 6: Power moves

## Before the session
- Run all three exec one-shots (links, images, spellcheck) on YOUR live site the day before - know the timing and what a real report looks like.
- Test a full cloud round-trip on your own repo: kick off, wait, review, merge. Know today's kickoff path from CLI, app, AND phone - you will demo the phone kickoff if time allows.
- Check your Plus window headroom morning-of; the session spends cloud tasks.
- Rehearse the lid-close beat: know how long your 404-page cloud task actually takes so the reopen timing works theatrically.
- Have the GitHub Action prompt pre-tested so Demo 3's yml is known-good.

## Timing (45 min)
- 0-3: Welcome - "the site works; now make it work without you"
- 3-9: Part 1 - codex exec concept + the three one-shots
- 9-17: Demo 1 - exec health check on their live sites
- 17-23: Part 2 - cloud tasks, three doors in, cloud-vs-local
- 23-33: Demo 2 - delegate the 404 page, close the lid, merge
- 33-38: Demo 3 - write the weekly Action, commit it
- 38-42: Part 3 speed-run - MCP, resume, plugins, the 10-min playbook
- 42-45: Part 4 send-off + graduation quiz

## Demo run-of-show
- Demo 1: exec finds a real broken link on someone's live site almost every time - use a volunteer's finding, fix it interactively, push. "exec finds, interactive fixes."
- Demo 2 key beat: closing the laptop lid while the cloud works. Be theatrical - close it, count ten with the room, talk over the closed laptop about what is happening in the cloud right now. This is the finale's showpiece; never cut it, never rush it.
- Demo 3: Codex writes the Action, the room reads it line by line with Codex explaining, commit. Enabling is homework - say so plainly.

## Watch for
- Plus window limits if the whole room fires cloud tasks simultaneously - STAGGER the kickoffs (rows, or odd/even birthdays). A blocked task mid-demo deflates the showpiece.
- Cloud task duration variance: if a task runs long past the lid-reopen, narrate the review of YOUR pre-run task while theirs finish.
- MCP rabbit holes - it gets one card and one example today; point the curious at the docs.

## Cuts if long
- Part 3 compresses to the playbook table alone (MCP and plugins become self-study pointers).
- Demo 3 can become "here is the prompt, run it tonight" homework.
- Never cut the lid moment or the send-off - people remember how courses end.
