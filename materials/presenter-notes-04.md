# Presenter notes - Session 4: Design and content

## Before the session
- Chase the notes file TWO DAYS ahead: attendees must bring their raw material (bio, things built, ideas) as plain text. Without it this session has nothing to build from. Send a reminder with a 5-minute template.
- Have a fallback notes file for anyone who arrives empty-handed - generic but realistic, so they can still follow the build.
- Run one full about-page build on your own site the day before so you know current output quality and timing.
- Have an accessibility checker ready to demo (or just Codex's own a11y audit prompt, pre-tested).

## Timing (45 min)
- 0-3: Welcome, show a finished three-page example (yours)
- 3-8: The pattern - notes in, pages out, skills doing the styling
- 8-22: Demo 1 - build about + showcase + ideas from their notes file
- 22-33: Demo 2 - polish loops (spacing, hierarchy, mobile)
- 33-41: Demo 3 - a11y audit and fixes
- 41-45: Homework + Q&A

## Demo run-of-show
- Demo 1: three pages from one notes file, with $site-design-system and $content-voice firing silently - callback to Session 3's trigger test.
- Demo 1 key beat: the "it exaggerated my bio" integrity moment. Codex will inflate someone's two-line bio into thought-leader prose. Stop everything when it happens. Read both versions aloud. Fix it with a boundary: "use only facts from my notes, do not embellish." This is the most important AI lesson in the course - never cut it, and if it does not happen naturally, plant it with your own bio.
- Demo 2: polish loops are Session 2's iteration laps at higher taste level. Two laps minimum.
- Demo 3: run the a11y audit, fix the top two findings. Contrast failures are the usual catch.

## Watch for
- The embellishment problem (above) - it WILL happen to several people. Make it a group moment, not a private fix.
- Content paralysis: people editing their bio for 10 minutes instead of building. Timebox: "ugly words now, beautiful pages today, better words next week."
- Mobile neglect - force one phone-width check per person in Demo 2.

## Cuts if long
- Demo 3 (a11y) compresses to "run this audit prompt at home" homework with the findings walkthrough next session.
- Showcase and ideas pages can be one-lap builds; the about page gets the full polish treatment.
- Never cut the integrity moment or the mobile check.
